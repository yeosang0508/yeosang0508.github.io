개발언어 및 프레임워크

  • 4. UI/UX 설계:
    • 사용자 친화적인 인터페이스와 직관적인 UI 설계를 통해 일정을 한눈에 볼 수 있도록 구현했습니다.

효율적인 설계의 중요성을 깨닫게 된 과정

프로젝트를 진행하며 RESTful 설계를 적용하지 못한 URL 구조와 데이터 처리의 비효율성을 경험했습니다. 당시 시간 부족과 RESTful 아키텍처에 대한 이해 부족으로 기존 방식을 유지했지만, 프로젝트를 마친 후 이 부분이 시스템 확장성과 유지보수에 큰 영향을 미친다는 사실을 깨달았습니다.

  • 1. URL 설계와 HTTP 메서드의 중요성:
    • 리소스를 중심으로 설계된 URL과 HTTP 메서드(GET, POST, PUT, DELETE)의 올바른 활용이 시스템 확장성과 유지보수성에 중요하다는 점을 인식했습니다.
  • 2. 시간 관리와 우선순위 설정:
    • 설계와 구현 사이의 균형을 잡지 못했던 경험을 바탕으로, 초기 설계 단계에 충분한 시간을 할애해야 한다는 교훈을 얻었습니다.
  • 3. 학습과 성장:
    • RESTful 아키텍처의 중요성을 학습하며, 이후 프로젝트에서 이를 적용하기 위해 꾸준히 연구하고 개인 프로젝트에 도입할 계획을 세웠습니다.
