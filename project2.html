TailsRoute
2024.10.02 ~ 2024.12.12

반려견의 건강, 식단, 운동 관리를 포함한 생활 전반을 체계적으로 관리하고, AI 기반 분석을 통해 맞춤형 정보를 제공하는 것을 목표로 하고 있습니다.

개발언어 및 프레임워크

팀 구성 및 역할

팀 구성은 5명이며, 팀장으로서 프로젝트를 주도하며 기획, 설계, 구현, 배포까지 전반적인 과정을 이끌었습니다.

기여 및 역할

  • 1. UI/UX 설계
    • Figma를 사용해 메인 페이지와 주요 UI 요소를 설계하고 프로토타입을 제작했습니다.
    • 사용자 경험을 고려하여 직관적이고 깔끔한 인터페이스를 설계했습니다.
  • 2. 기능 구현
    • 로그인 및 인증 기능: 로컬 로그인, 카카오 로그인, 이메일 인증 기능을 개발하며 사용자 인증을 효율적으로 처리했습니다.
    • 데이터 처리: JSON 데이터를 전처리하고, 대량의 데이터를 처리하기 위해 Google Drive를 활용해 데이터를 효율적으로 관리했습니다. CSV 파일 생성 및 통합 작업을 수행하며, 시각화를 통해 데이터 품질을 검증했습니다.
    • AI 모델: ST-GCN을 활용해 행동 분석 모델을 학습했으며, MobileNet을 사용해 객체를 감지하고 키포인트를 추출해 모델 입력 데이터를 생성했습니다. 결과가 완벽하지는 않았지만, 객체 감지 기능은 성공적으로 구현했습니다.
  • 3. 프로젝트 관리 및 협업
    • GitHub (Develop 브랜치)를 사용해 팀원들과 협업하며, 각 기능 개발 완료 후 코드에 대한 코멘트를 남겨 코드 품질을 개선했습니다.
    • 코드 충돌로 인한 어려움이 있었으나, 팀원들과 지속적으로 소통하며 문제를 해결했습니다.
    • Notion을 활용해 팀원들의 작업 진행 상황을 관리하고, 일정과 우선순위를 조율했습니다.

문제점 및 해결 과정

  • 1. 그래픽 카드 성능 부족
    • Keypoint R-CNN 모델을 활용하고자 했으나, 그래픽카드 성능의 한계로 MobileNet 모델을 사용했습니다.
    • MobileNet 모델은 경량화된 구조로 인해 성능이 낮아 객체 감지와 키포인트 추출에서 성능 저하를 경험했습니다.
    • 성능 한계 내에서 데이터를 최적화하고 모델 학습 과정을 반복하며 결과를 개선하려 노력했습니다.
    • 이 과정을 통해 하드웨어 자원의 중요성과 향후 프로젝트에서 적절한 환경 설계의 필요성을 배웠습니다.
  • 2. 데이터 처리의 어려움
    • 대량의 JSON 데이터를 관리하고 처리하는 데 시간이 많이 소요되었습니다.
    • Google Drive를 활용해 데이터를 정리하고 공유하며, 데이터 통합 작업을 효율적으로 수행했습니다.
  • 3. 코드 충돌 문제
    • GitHub 협업 과정에서 코드 충돌 문제가 발생해 이를 해결하기 위한 방안을 논의했습니다.
    • 팀원들의 작업이 병합되는 과정에서 충돌이 빈번하게 발생하며, Develop 브랜치 방식을 도입해 협업 프로세스를 개선했습니다.
    • 각 팀원은 자신의 작업을 Feature 브랜치에서 수행한 후, 변경 내용을 공유하고 검토한 뒤 Develop 브랜치로 병합하도록 작업 방식을 재정립했습니다.
    • 이를 통해 충돌을 최소화하고 작업 효율을 높일 수 있었습니다.
